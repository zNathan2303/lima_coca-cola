1. O que é um repositório no Git e qual a sua importância?

O Git é uma forma de criar um repositório local que pode ser compartilhado com outras pessoas por meio do Github. Ele é importante pois permite criar várias "linhas do tempo", permitindo que tenha várias pessoas trabalhando de forma simultânea em um mesmo projeto.
 
2. Qual a função do Git Bash?

Ele permite a criação de repositórios Git e a utilização dos comandos para realizar o commit do projeto e mandá-lo para o Github.

3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?

O comando git add serve para permitir que as modificações sejam salvas para realizar o commit. O git add <arquivo> permite apenas as alterações de um determinado arquivo serem utilizadas em um commit, já o git add . faz com que todas as alterações possam ser incluidas no próximo commit.

4. Por que utilizamos o ponto (.) no comando git add .?

Para incluir todos arquivos da pasta de onde estamos.

5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.

O comando git commit serve para registrar uma alteração relevante no código, sendo importante descrever as alterações que foram feitas para saber como está o andamento do projeto.

6. Qual a função do comando git push?

Ele envia o commit feito no repositório local para o repositório remoto(Github).

7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?

Para evitar possíveis perdas de mudanças relevantes por causa de imprevistos ou então retomar o projeto de um ponto aonde estava tudo funcionando perfeitamente.

8. Explique a função das tags HTML e como elas estruturam uma página.

Elas servem para definir o "corpo" da página, em que terá informações da página que ficam ocultas, e outras que os usuários poderâo ver, sejam caixas(como divs), texto(h1, span, p, etc.), links ou imagens.

9. O que são atributos em uma tag HTML e qual a sua função?

Eles são textos colocados antes de fechar a tag HTML de abertura, e servem para atribuir características especiais para ela, permitindo outros funcionamentos. 

10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?

Ele é uma forma de fazer abreviações na criação de estruturas HTML. E ele facilita o desenvolvimento de páginas HTML pois permite os desenvolvedores pouparem tempo criando estruturas muito grandes apenas por simples comandos.

11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?

O box-sizing é uma forma de definir como será calculado a altura e a largura de algo. Ele é utilizado no reset inicial pois permite um melhor controle do desenvolvedor em relação ao CSS, definindo o box-sizing logo no começo para todos os itens que forem sendo adicionados ao longo do tempo.

12. Por que é importante utilizar tags semânticas no HTML?

Para melhorar o entendimento de onde está localizado cada parte da página, facilitando a visualização principalmente para outras pessoas que estejam trabalhando no mesmo projeto.

13. O que são media queries no CSS e qual a sua função?

Eles são regras usadas para ajustar o site com base na tela em que ele está sendo aberto. A função dele é justamente permitir a melhor visualização do site independente do dispositivo que está abrindo.

14. Qual a função do GitHub Pages?

Permitir a visualização do site de maneira remota, sem a necessidade de baixar os arquivos para iniciar a visualização a partir do dispositivo.

15. O que é o Flexbox no CSS e para que ele é utilizado?

O flexbox é um modelo de layout dinâmico. Ele é usado principalmente para tornar o layout do site mais flexível, permitindo que seja facilmente ajustável em qualquer tamanho de tela.

16. Qual a diferença entre justify-content e align-items em um container Flexbox?

O justify-content serve para alinhar os itens de forma horziontal (por padrão) e o align-items alinha de forma vertical (por padrão). O padrão de alinhamento é row.

17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?

O flex-grow serve para expandir um objeto o máximo possível que estiver disponível, colocando nele um número da "força" de expansão dele. No layout flexbox ele serve para facilitar o ajuste do tamanho dos objetos, utilizando o número que representa a "força" para deixar o tamanho de algum objeto maior que outro.

18. Explique a diferença entre flex-direction: column; e flex-direction: row;.

O flex-direction: row é o que vem por padrão, nele os objetos flex se organizam da esquerda para direita, em forma de linha. Já no flex-direction: column, os objetos se organizam de cima para baixo, como se fosse uma coluna. Caso o flex-direction seja definido como column, as propriedades do justify-content e align-items são invertidas.
