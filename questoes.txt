1. O que é um repositório no Git e qual a sua importância?

O repositório Git é um repositório local que pode ser compartilhado com outras pessoas por meio do Github. Ele é importante pois permite criar várias "linhas do tempo", permitindo que tenha várias pessoas trabalhando de forma simultânea em um mesmo projeto.
 
2. Qual a função do Git Bash?

Ele permite a criação de repositórios Git e a utilização dos comandos para realizar o commit e mandá-los para o Github.

3. O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?

O comando git add serve para permitir as modificações serem salvas para realizar o commit. O git add <arquivo> permite apenas as alterações de um determinado arquivo serem utilizadas em um commit, já o git add . faz com que todas as alterações possam ser incluidas no próximo commit.

4. Por que utilizamos o ponto (.) no comando git add .?

Para incluir todos arquivos dentro da pasta de onde estamos.

5. Explique o comando git commit e por que é importante criar uma mensagem descritiva.

O comando git commit serve para registrar uma alteração relevante no código, sendo importante descrever as alterações que foram feitas para saber como está o andamento do projeto.

6. Qual a função do comando git push?

Ele envia o commit feito no repositório local para o repositório remoto(Github)

7. Qual a importância de criar commits frequentes durante o desenvolvimento de um projeto?

Para evitar possíveis perdas de mudanças relevantes por causas de imprevistos ou então retomar o projeto de um ponto aonde estava tudo funcionando perfeitamente.

8. Explique a função das tags HTML e como elas estruturam uma página.

Elas servem para definir os objetos que estarão na página, sejam caixas(como divs) ou texto(h1, span, p, etc.).

9. O que são atributos em uma tag HTML e qual a sua função?

Eles são caracteristicas especiais que podem ser atribuidas para as tags HTML, como por exemplo atribuir um nome para elas(id).

10. O que é o Emmet e como ele facilita o desenvolvimento de páginas HTML?

Ele é uma forma de atalho para agilizar o desenvolvimento de páginas, permitindo que uma grande quantidade de código (estrutura HTML) possa ser escrita de forma rápida (! + tecla tab).

11. O que é o box-sizing no CSS e por que usamos ele no reset inicial?

Permite ajustar o tamanho e largura máxima de algum elemento, incluindo não só o tamanho dele, mas margens e outras configurações que possam acabar mexendo no tamanho total dele. Ele é usado no reset inicial pois permite padronizar todos os elementos de uma só vez.

12. Por que é importante utilizar tags semânticas no HTML?

Para melhorar o entendimento do código, permitindo identificar estruturas específicas e onde elas estão localizadas.

13. O que são media queries no CSS e qual a sua função?

Os media queries são formas de formatação da página que são ativadas quando, por exemplo, o tamanho da tela está sendo exibido. A sua função é ajustar a visualização do site independente do tamanho da tela em que está sendo exibido.

14. Qual a função do GitHub Pages?

Permitir a visualização do site sem a necessidade de baixar os arquivos.

15. O que é o Flexbox no CSS e para que ele é utilizado?

É uma forma de ajustar o display para deixar os elementos da página dinâmicos.

16. Qual a diferença entre justify-content e align-items em um container Flexbox?

O justify-content ajusta o conteúdo na horizontal, e o align-items ajusta na vertical. Ambos são assim no flex-derection padrão(row).

17. Como o flex-grow funciona e qual o seu propósito no layout Flexbox?

Ele faz com que o elemento se expanda o máximo possível, podendo atribuir uma "força" para a expansão.

18. Explique a diferença entre flex-direction: column; e flex-direction: row;.

O flex-direction: column faz com que os elementos se alinhem de forma vertical, de cima para baixo. Já o flex-direction: row é a forma padrão em que vêm, em que os elementos se alinham de forma horizontal, da esquerda para direita(forma padrão).
